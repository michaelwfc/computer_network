Example — Simplified Handshake Pseudocode
state = "CLOSED"

on connect():
    send(SYN)
    state = "SYN_SENT"

on receive(SYN_ACK):
    send(ACK)
    state = "ESTABLISHED"

This pseudocode shows how TCP transitions from a closed state into an established connection after completing the handshake.

TCP also supports a rare scenario called simultaneous open, where both sides send SYN packets at the same time. Each side then acknowledges the other's SYN.

Key Takeaway: The three-way handshake establishes mutual agreement, synchronizes sequence numbers, and confirms bidirectional communication.
Section 04

Connection Teardown

Ending a TCP connection is more complicated than ending a phone call. TCP must ensure that all in-flight data has arrived before deleting connection state.

TCP uses the FIN flag to indicate that one side has no more data to send.

Typical Teardown Sequence
  1. A → B: FIN
  2. B → A: ACK
  3. B → A: FIN
  4. A → B: ACK

Notice something important: TCP connections are full duplex. Each direction of communication shuts down independently.

One side may finish sending data while still receiving data from the other side.

This is similar to two people talking on a radio. One person may say “I'm done talking,” but still continue listening.

Example — Closing a TCP Connection
on close():
    send(FIN)
    state = "FIN_WAIT_1"

on receive(ACK):
    state = "FIN_WAIT_2"

on receive(FIN):
    send(ACK)
    state = "TIME_WAIT"

This logic demonstrates how a TCP endpoint safely transitions through shutdown states while acknowledging the remote side.

Key Takeaway: TCP teardown is symmetric. Both sides must independently signal that they are finished sending data.
